# 01-工具接口设计

工具接口设计是 Agent 与外部系统交互的基础。本文档详细介绍 OpenAI Tool Calling 格式、函数定义规范以及工具描述的最佳实践。

## OpenAI Tool Calling 格式

### 工具定义结构

```json
{
  "type": "function",
  "function": {
    "name": "tool_name",
    "description": "工具功能的详细描述",
    "parameters": {
      "type": "object",
      "properties": {
        "param_name": {
          "type": "string",
          "description": "参数的用途说明"
        }
      },
      "required": ["param_name"]
    }
  }
}
```

### 完整示例

```json
{
  "type": "function",
  "function": {
    "name": "search_code",
    "description": "在代码库中搜索指定的符号、函数或文本。支持正则表达式搜索。",
    "parameters": {
      "type": "object",
      "properties": {
        "query": {
          "type": "string",
          "description": "搜索关键词或正则表达式"
        },
        "file_pattern": {
          "type": "string",
          "description": "文件过滤模式，如 *.py, src/**/*.js"
        },
        "scope": {
          "type": "string",
          "enum": ["all", "definitions", "references"],
          "description": "搜索范围：全部、定义处、引用处",
          "default": "all"
        },
        "max_results": {
          "type": "integer",
          "description": "最大返回结果数",
          "default": 20
        }
      },
      "required": ["query"]
    }
  }
}
```

## 函数定义规范

### 1. 工具命名规范

- 使用 snake_case 命名（如 `search_code`）
- 名称应清晰表达功能
- 避免过于通用或模糊的名称

### 2. 描述编写最佳实践

**好的描述示例：**

```json
{
  "description": "读取指定文件的内容。支持部分读取（指定行号范围）和全部读取。"
}
```

**需要改进的描述：**

```json
{
  "description": "读取文件"  // 过于简略
}
```

### 3. 参数定义规范

#### 必须指定类型的参数

```json
"properties": {
  "path": {
    "type": "string",
    "description": "文件路径，绝对路径或相对于工作目录的路径"
  },
  "line_start": {
    "type": "integer",
    "description": "起始行号（1-indexed）"
  },
  "line_end": {
    "type": "integer",
    "description": "结束行号（包含）"
  }
}
```

#### 使用 enum 限制可选值

```json
"operation": {
  "type": "string",
  "enum": ["create", "read", "update", "delete"],
  "description": "操作类型"
}
```

#### 设置合理的默认值

```json
"max_results": {
  "type": "integer",
  "description": "最大返回结果数",
  "default": 20,
  "minimum": 1,
  "maximum": 100
}
```

## 复杂参数结构

### 嵌套对象参数

```json
{
  "type": "function",
  "function": {
    "name": "glob_files",
    "description": "根据模式匹配文件",
    "parameters": {
      "type": "object",
      "properties": {
        "patterns": {
          "type": "array",
          "items": {
            "type": "string"
          },
          "description": "glob 模式列表"
        },
        "options": {
          "type": "object",
          "properties": {
            "exclude_dirs": {
              "type": "array",
              "items": {"type": "string"},
              "description": "要排除的目录"
            },
            "include_hidden": {
              "type": "boolean",
              "description": "是否包含隐藏文件",
              "default": false
            }
          },
          "description": "匹配选项"
        }
      },
      "required": ["patterns"]
    }
  }
}
```

### 数组参数

```json
{
  "type": "function",
  "function": {
    "name": "batch_read_files",
    "description": "批量读取多个文件",
    "parameters": {
      "type": "object",
      "properties": {
        "paths": {
          "type": "array",
          "items": {"type": "string"},
          "description": "要读取的文件路径列表"
        }
      },
      "required": ["paths"]
    }
  }
}
```

## 工具描述的 Prompt 工程

### 1. 让 LLM 准确理解工具用途

```json
{
  "description": "在项目代码库中搜索指定的函数、类或变量。返回定义位置、调用位置和相关代码片段。"
}
```

### 2. 说明使用场景

```json
{
  "description": "搜索代码库中的符号。当你需要：1) 查找函数定义位置；2) 了解某个符号的所有使用处；3) 快速定位相关代码时使用此工具。"
}
```

### 3. 明确输入输出格式

```json
{
  "description": "列出目录中的文件和子目录。返回文件列表，包含每个文件的名称、大小和修改时间。"
}
```

## 多工具协作设计

### 工具分组

```python
# 文件操作工具组
file_tools = [
    {
        "type": "function",
        "function": {
            "name": "read_file",
            "description": "读取文件内容...",
            "parameters": {...}
        }
    },
    {
        "type": "function",
        "function": {
            "name": "write_file",
            "description": "写入或创建文件...",
            "parameters": {...}
        }
    },
    {
        "type": "function",
        "function": {
            "name": "edit_file",
            "description": "修改文件内容...",
            "parameters": {...}
        }
    }
]

# 代码搜索工具组
search_tools = [
    {
        "type": "function",
        "function": {
            "name": "search_code",
            "description": "在代码库中搜索...",
            "parameters": {...}
        }
    },
    {
        "type": "function",
        "function": {
            "name": "find_definitions",
            "description": "查找符号定义...",
            "parameters": {...}
        }
    }
]
```

### 工具优先级

有时需要设置工具的优先级，引导 LLM 优先使用某些工具：

```python
def get_tools_with_priority():
    # 高优先级工具（更常用）
    high_priority = [read_file, search_code]
    # 普通优先级
    normal_priority = [glob_files, find_definitions]
    # 低优先级工具
    low_priority = [execute_command, web_search]

    return high_priority + normal_priority + low_priority
```

## 最佳实践总结

| 实践 | 说明 |
|------|------|
| 描述要详细 | 包含功能、用途、使用场景 |
| 参数类型明确 | 始终指定 type，使用 enum 限制枚举值 |
| 合理设置默认值 | 减少必填参数，降低调用难度 |
| 名称要清晰 | 使用动词+名词的命名方式 |
| 描述输入格式 | 让 LLM 知道如何构造参数 |
| 分组管理 | 相关工具放在一起，便于理解 |
