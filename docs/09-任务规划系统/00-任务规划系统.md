# 09-任务规划系统

任务规划系统是 Agent 具备"思考"能力的关键，使其能够将复杂任务分解为可执行的步骤。本模块详细介绍分层规划、任务拆解、状态机和长任务稳定性。

## 模块目录

- [01-分层规划架构](./01-分层规划架构.md) - 战略/战术/执行三层规划
- [02-任务拆解方法](./02-任务拆解方法.md) - 自顶向下、分而治之
- [03-状态机实现](./03-状态机实现.md) - Agent 状态管理、循环控制
- [04-长任务稳定性](./04-长任务稳定性.md) - 断点续做、错误恢复

## 核心概念

### 任务规划系统架构

```
┌─────────────────────────────────────────────────────────────┐
│                    任务规划系统架构                           │
├─────────────────────────────────────────────────────────────┤
│  ┌─────────────────────────────────────────────────────┐   │
│  │                   战略层 (Strategic)                 │   │
│  │   理解目标 → 制定计划 → 评估可行性                   │   │
│  └─────────────────────────────────────────────────────┘   │
│                           ↓                                  │
│  ┌─────────────────────────────────────────────────────┐   │
│  │                   战术层 (Tactical)                  │   │
│  │   任务拆解 → 步骤排序 → 依赖管理                     │   │
│  └─────────────────────────────────────────────────────┘   │
│                           ↓                                  │
│  ┌─────────────────────────────────────────────────────┐   │
│  │                   执行层 (Execution)                │   │
│  │   工具调用 → 结果验证 → 迭代优化                     │   │
│  └─────────────────────────────────────────────────────┘   │
└─────────────────────────────────────────────────────────────┘
```

### 核心公式

```
Planning = Goal Decomposition + Step Sequencing + Execution + Adaptation
```

## 关键知识点

| 知识点 | 说明 |
|--------|------|
| 任务分解 | 将复杂任务拆分为子任务 |
| 依赖管理 | 处理任务间的依赖关系 |
| 状态跟踪 | 跟踪任务执行状态 |
| 错误恢复 | 失败后的重试和回退 |
| 动态调整 | 根据执行结果调整计划 |
