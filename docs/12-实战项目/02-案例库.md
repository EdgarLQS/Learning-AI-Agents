## 1. 代码理解类

### 案例 1.1: 理解项目结构

**用户请求**:
```
"这是一个什么项目？它的主要功能是什么？"
```

**Agent 执行**:
```
1. 读取 README.md
2. 扫描项目结构
3. 查找入口文件 (main.py, index.js, app.py 等)
4. 分析核心模块
```

**输出示例**:
```
这是一个 Python Web 项目，使用 Flask 框架。
主要功能：
- 用户认证
- API 接口
- 数据存储

核心文件：
- app.py: 主应用
- models/: 数据模型
- routes/: 路由定义
```

### 案例 1.2: 查找相关代码

**用户请求**:
```
"找出所有处理用户登录的代码"
```

**Agent 执行**:
```
1. 搜索关键词 "login", "authenticate", "sign in"
2. 读取相关文件
3. 整理调用链
```

**关键代码位置**:
```
- routes/auth.py: 登录 API
- services/auth_service.py: 认证逻辑
- models/user.py: 用户模型
```

### 案例 1.3: 分析依赖关系

**用户请求**:
```
"这个函数被哪些地方调用？"
```

**Agent 执行**:
```
1. 解析 AST 找到函数定义
2. 搜索所有函数调用
3. 构建调用图
```

## 2. 代码修改类

### 案例 2.1: 修复 Bug

**问题描述**:
```
用户头像上传失败，大于 1MB 的文件被拒绝
```

**Agent 分析**:
```
1. 搜索 "avatar", "upload", "file size"
2. 找到 upload_utils.py 中的检查逻辑
3. 发现 max_size = 1024 (字节) 应该是 1024*1024
```

**修复代码**:
```python
# 修复前
MAX_SIZE = 1024  # bytes

# 修复后
MAX_SIZE = 1024 * 1024  # 1MB
```

### 案例 2.2: 添加功能

**需求**:
```
"为用户列表添加分页功能"
```

**Agent 执行**:
```
1. 查找用户列表 API
2. 理解现有实现
3. 添加分页参数
4. 更新响应格式
```

**修改位置**: `routes/users.py`

### 案例 2.3: 重构代码

**需求**:
```
"把这段重复的验证逻辑抽取成函数"
```

**原始代码**:
```python
# 验证 email
if '@' not in email:
    raise ValueError("Invalid email")

# 验证 phone
if '@' not in phone:  # 错误：复制粘贴错误
    raise ValueError("Invalid phone")
```

**重构后**:
```python
def validate_email(email):
    if '@' not in email:
        raise ValueError("Invalid email")

def validate_phone(phone):
    if len(phone) != 11:
        raise ValueError("Invalid phone")

# 使用
validate_email(user.email)
validate_phone(user.phone)
```

## 3. 测试相关类

### 案例 3.1: 编写测试用例

**需求**:
```
"为用户注册功能编写单元测试"
```

**Agent 生成的测试**:
```python
# tests/test_user_registration.py
import pytest
from app import create_user

def test_create_user_success():
    user = create_user("test@example.com", "password123")
    assert user.email == "test@example.com"
    assert user.password_hash is not None

def test_create_user_duplicate_email():
    create_user("test@example.com", "password123")
    with pytest.raises(Exception):
        create_user("test@example.com", "password456")

def test_create_user_invalid_email():
    with pytest.raises(ValueError):
        create_user("invalid", "password123")
```

### 案例 3.2: 修复测试失败

**错误信息**:
```
FAILED tests/test_api.py::test_get_user

```

**Agent 修复**:
```
1. 查看测试期望
2. 查找 API 返回值
3. 更新 API 或测试（根据需求）
```

## 4. 工程任务类

### 案例 4.1: 项目初始化

**需求**:
```
"创建一个新的 Python 项目，包含 Flask 和数据库"
```

**Agent 执行**:
```bash
# 1. 创建项目目录
mkdir myproject && cd myproject

# 2. 创建虚拟环境
python -m venv venv

# 3. 创建基础文件
touch app.py requirements.txt README.md

# 4. 安装依赖
pip install flask flask-sqlalchemy
```

### 案例 4.2: 依赖管理

**需求**:
```
"添加 pytest 作为开发依赖"
```

**执行**:
```bash
pip install pytest --dev
# 或
poetry add --group dev pytest
```

### 案例 4.3: 构建部署

**需求**:
```
"为这个项目创建 Dockerfile"
```

**生成的 Dockerfile**:
```dockerfile
FROM python:3.11-slim

WORKDIR /app

COPY requirements.txt .
RUN pip install -r requirements.txt

COPY . .

EXPOSE 8000

CMD ["python", "app.py"]
```

## 常见问题与解决

### Q1: Agent 陷入循环

**症状**: 重复执行相同的动作

**解决方案**:
- 添加循环检测
- 设置最大迭代次数
- 加入随机性

### Q2: 工具调用失败

**症状**: 文件不存在、权限错误

**解决方案**:
- 添加存在性检查
- 完善错误处理
- 提供友好的错误信息

### Q3: 修改不正确

**症状**: 生成的代码有语法错误

**解决方案**:
- 添加语法检查
- 使用 Linter
- 修改后验证

### Q4: 上下文超限

**症状**: 任务信息太多导致超出窗口

**解决方案**:
- 智能截取关键信息
- 使用 RAG 检索相关代码
- 分批处理

## 最佳实践

1. **小步迭代**: 先验证每步正确，再继续
2. **保留备份**: 修改前保存原文件
3. **及时验证**: 修改后运行测试
4. **清晰日志**: 记录每步操作
