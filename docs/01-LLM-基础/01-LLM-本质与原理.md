# LLM 本质与原理

## 1.1 LLM 本质

LLM (Large Language Model，大语言模型) 本质是一个**概率预测模型**，其核心任务是预测下一个 token（词元）出现的概率。

```
LLM 本质 = 输入文本 → 概率计算 → 预测下一个 token
```

### 核心公式

```
P(next_token | context) → 输出概率分布 → 选择 token
```

这个过程被称为 **Next Token Prediction（下一个词预测）**，是 LLM 训练和推理的基础。

## 1.2 工作原理

### 自回归生成

LLM 采用**自回归（Autoregressive）**方式生成文本：

1. 接收输入文本（prompt）
2. 计算下一个 token 的概率分布
3. 根据采样策略选择一个 token
4. 将选中的 token 加入输入，重复步骤 2-4

```
输入: "今天天气"
  ↓
模型输出概率分布
  ↓
选择: "很好" (35%)
  ↓
新的输入: "今天天气很好"
  ↓
继续预测下一个 token...
```

### 概率分布示例

给定输入 `"今天天气"`，模型可能输出：

| 候选 Token | 概率 |
|-----------|------|
| 很好 | 35% |
| 不错 | 28% |
| 晴朗 | 15% |
| 暖和 | 8% |
| 炎热 | 5% |
| ... | 9% |

模型并不"理解"语言，而是通过学习海量文本中的**统计规律**来预测下一个最可能出现的词。

## 1.3 语言建模的本质

### 统计语言模型

LLM 本质上是一个**统计语言模型**。它通过学习大量文本，捕捉：

- **词法规律**：哪些词经常一起出现
- **语法规律**：符合语法规则的句子概率更高
- **语义规律**：语义相似的词在相似上下文中出现
- **世界知识**：训练数据中包含的各种信息

### 压缩与泛化

LLM 可以看作是对训练数据的**有损压缩**：

- 训练：将海量文本的信息压缩到模型参数中
- 推理：从参数中"解压"出与输入相关的知识

这种压缩使模型获得了**泛化能力**，能够处理训练时从未见过的输入。

## 1.4 模型规模

### 参数数量

现代 LLM 的参数规模通常在数十亿到数千亿：

| 模型 | 参数规模 |
|------|----------|
| GPT-3 | 175B |
| GPT-4 | ~1.8T (MoE) |
| LLaMA 3 70B | 70B |
| Claude 3 | 未公开 |

### 涌现能力

当模型规模超过某个阈值时，会出现**涌现能力（Emergent Abilities）**：

- 思维链推理（Chain-of-Thought）
- 零样本学习（Zero-shot Learning）
- 指令跟随（Instruction Following）

## 1.5 LLM 与 AI Agent

```
LLM 本身 = 被动回答问题

LLM + Agent = 主动执行任务

AI Agent = LLM + Planning + Tools + Memory
```

LLM 是 AI Agent 的**核心引擎**，提供了理解、推理和生成能力。Agent 在 LLM 基础上增加了：
- **规划（Planning）**：分解任务、制定步骤
- **工具（Tools）**：调用外部系统执行操作
- **记忆（Memory）**：存储和检索上下文信息

## 常见问题

### Q: LLM 是如何"理解"语言的？

LLM 并不像人类一样"理解"语言。它通过学习海量文本中的统计规律，建立了 token 之间的关联关系。当输入一个 prompt 时，模型根据这些统计规律计算下一个 token 的概率分布。

### Q: 为什么 LLM 有时会生成错误信息？

因为 LLM 本质是概率模型，它生成的内容取决于训练数据中的统计规律，而非事实准确性。这导致了"幻觉"（Hallucination）问题——生成的内容看似合理但实际上是错误的。

### Q: LLM 和搜索引擎有什么区别？

- **搜索引擎**：从索引的网页中检索相关信息
- **LLM**：根据训练数据中的统计规律生成内容

LLM 生成的内容是"虚构"的，而搜索引擎返回的是真实存在的信息。
