# 01-规划能力不足

规划能力不足是 Agent 面临的首要挑战。本文档详细分析规划失败的原因并提供改进策略。

## 规划失败原因

```
┌─────────────────────────────────────────────────────────────────┐
│                     规划失败的原因                               │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│  1. 任务理解不准确                                              │
│     • 目标模糊                                                  │
│     • 隐含需求未识别                                           │
│     • 领域知识缺乏                                              │
│                                                                  │
│  2. 步骤拆解不完整                                              │
│     • 遗漏必要步骤                                              │
│     • 步骤间依赖关系错误                                        │
│     • 粒度不一致                                                │
│                                                                  │
│  3. 执行过程偏移                                               │
│     • 中途发现新信息                                            │
│     • 初始假设错误                                              │
│     • 环境变化                                                  │
│                                                                  │
│  4. 错误恢复困难                                               │
│     • 缺乏回退机制                                              │
│     • 难以识别错误点                                            │
│     • 重试策略不当                                              │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

class IterativePlanner:
    """迭代式规划器"""

    async def plan_with_iteration(self, goal: str, context: dict) -> dict:
        """迭代规划"""

        plan = await self._initial_plan(goal, context)
        iteration = 0
        max_iterations = 3

        while iteration < max_iterations:
            # 验证计划
            issues = await self._validate_plan(plan, context)

            if not issues:
                break

            # 改进计划
            plan = await self._improve_plan(plan, issues)
            iteration += 1

        return plan
